# 特殊道具系统设计文档

## 1. 文档目标

本文档定义当玩家在高难关卡中连续失败、出现明显卡关时，如何通过“特殊道具”提高过关概率，同时保证：

- 不破坏主玩法
- 不削弱解题乐趣
- 不让玩家感觉系统在替自己玩

本文档覆盖：

- 救场型道具设计原则
- 当前版本推荐道具列表
- 道具 JSON 配置字段设计
- 连败保护与失败补偿规则

## 2. 设计原则

### 2.1 系统目标

这套系统不是“直接帮玩家过关”，而是：

- 给玩家一次翻盘机会
- 降低极端卡关带来的挫败感
- 让困难局面出现更多可选解

### 2.2 核心原则

- 道具只改变“机会”，不直接给出答案
- 道具应优先保留瞄准、反弹、结构选择的乐趣
- 道具效果必须一眼能理解
- 道具最好只影响一小段局面，而不是彻底跳过关卡

### 2.3 不建议的方向

首版不建议加入：

- 一键清屏
- 自动完成收集
- 自动通过关卡
- 完全替玩家选择发射位置

## 3. 当前推荐道具体系

结合当前玩法，推荐首版采用以下 5 种救场型道具：

- `换球`
- `彩虹道具球`
- `炸裂道具球`
- `破障锤`
- `缸体吸附增强`

这 5 种基本覆盖了当前最常见的卡关原因：

- 颜色不对
- 关键位置差一色
- 结构被障碍卡住
- 石头/冰冻难处理
- 收集差一点完成

## 4. 道具详细设计

## 4.1 换球

### 定位

最基础的修正型道具，用于处理“当前发射球颜色不理想”的情况。

### 功能规则

- 使用后，将 `当前发射球` 与 `下一发预览球` 交换
- 如果没有下一发特殊球，则只在普通球之间交换

### 适用场景

- 关键挂点差一色
- 当前球明显无用
- 玩家已经看出解法，但只缺颜色

### 优点

- 实现简单
- 认知成本低
- 不破坏结构规则

### 平衡建议

- 单局使用次数建议 `1`
- 失败保护赠送优先级最高

## 4.2 彩虹道具球

### 定位

直接把当前发射球改成彩虹球，用于解决“差一个颜色就能起飞”的局面。

### 功能规则

- 使用后，当前发射球临时变为彩虹球
- 彩虹球规则复用 [SKILL_BALLS_SPEC.md](E:\cocos_project\bubble\SKILL_BALLS_SPEC.md) 中的 `彩虹技能球`

### 适用场景

- 关键支撑点差一色
- 小缝挂点差一色
- 高价值掉落前差一步

### 平衡建议

- 强于换球，频率应低于换球
- 单局建议不超过 `1`

## 4.3 炸裂道具球

### 定位

结构破坏型道具，用于处理石头障碍、冰冻外壳、封路点和桥接点。

### 功能规则

- 使用后，当前发射球临时变为炸裂球
- 炸裂球规则复用 [SKILL_BALLS_SPEC.md](E:\cocos_project\bubble\SKILL_BALLS_SPEC.md) 中的 `炸裂技能球`

### 适用场景

- 石头球挡路
- 冰冻球卡关键位置
- 结构过于封闭
- 需要快速打开一个通道

### 平衡建议

- 强度高于彩虹球
- 不建议和破障锤在同局大量叠加

## 4.4 破障锤

### 定位

定点处理障碍的道具，用于精准缓解“我知道问题在哪，但够不到”的挫败。

### 功能规则

- 玩家选择一个障碍球目标
- 如果目标是 `stone`：
  - 直接清除该石头障碍球
- 如果目标是 `ice`：
  - 直接解冻，转化为其 `innerColor` 对应的普通球

### 限制规则

- 不能作用于普通颜色球
- 不能作用于技能球

### 适用场景

- 关卡被单个障碍点卡死
- 关键通路被石头或冰冻球锁住

### 平衡建议

- 强度高，但趣味性偏工具型
- 单局建议只允许 `1`

## 4.5 缸体吸附增强

### 定位

这是最贴合本项目下半场玩法的特色道具，用于缓解“上方打得不错，但下方总差一点收集”的情况。

### 功能规则

开启后，持续若干回合或若干次掉落结算：

- 缸体中心有效收集区临时变大
- 或掉落珠在接近缸中心时增加轻微吸附

首版建议选择其中一种，不要两种都开。

### 推荐首版方案

- `collectZoneScaleTemporary +0.15 ~ +0.25`

### 适用场景

- 收集目标较高
- 玩家上方操作已经达标，但下方接取总差一点

### 平衡建议

- 这是辅助型道具，不应直接影响上方消除规则
- 更适合失败补偿，而不是常驻主力道具

## 5. 推荐优先级

如果首版只做 3 种，建议优先做：

1. `换球`
2. `彩虹道具球`
3. `炸裂道具球`

如果做 5 种完整救场体系，再补：

4. `破障锤`
5. `缸体吸附增强`

## 6. 道具来源设计

推荐分三类来源：

### 6.1 局前携带

- 玩家进入关卡前可带入 `1~2` 个道具

### 6.2 连败补偿

- 连续失败若干次后，系统给予推荐道具

### 6.3 活动/奖励发放

- 登录奖励
- 活动掉落
- 新手赠送

首版如果要控制复杂度，优先做：

- 连败补偿

## 7. 失败保护与连败补偿

### 7.1 目标

当玩家明显卡关时，逐步提高其翻盘机会，而不是一次性把关卡变简单。

### 7.2 推荐规则表

```json
{
  "failureProtection": [
    {
      "failCount": 2,
      "offer": "swap_ball",
      "freeUses": 1
    },
    {
      "failCount": 3,
      "offer": "rainbow_ball",
      "freeUses": 1
    },
    {
      "failCount": 4,
      "offer": "blast_ball",
      "freeUses": 1
    },
    {
      "failCount": 5,
      "offer": "jar_magnet_boost",
      "freeUses": 1
    }
  ]
}
```

### 7.3 推荐实际节奏

- 连败 `2` 次：推荐 `换球`
- 连败 `3` 次：推荐 `彩虹道具球`
- 连败 `4` 次：推荐 `炸裂道具球`
- 连败 `5` 次：推荐 `缸体吸附增强`

如果关卡障碍非常重，也可以把第 5 次补偿替换成：

- `破障锤`

### 7.4 保护机制边界

- 连败补偿只作用于当前关卡
- 过关后连败计数清零
- 每次只推荐 1 种主道具，避免信息过多

## 8. 动态轻辅助建议

除了显性道具，还可以加入轻量动态辅助，但要克制。

可选项：

- 连败后略微提高当前颜色命中概率
- 连败后降低无效球概率
- 连败后略微放宽缸中心判定

注意：

- 这些辅助不建议明示给玩家
- 幅度必须小，避免让玩家觉得系统在操控胜负

## 9. JSON 配置字段设计

## 9.1 关卡内临时可用道具

推荐在关卡配置中增加：

```json
"powerupRules": {
  "allowedPowerups": [
    "swap_ball",
    "rainbow_ball",
    "blast_ball",
    "barrier_hammer",
    "jar_magnet_boost"
  ],
  "failureProtectionEnabled": true
}
```

字段说明：

- `allowedPowerups`
  - 本关允许使用的道具列表

- `failureProtectionEnabled`
  - 是否启用连败保护

## 9.2 全局道具定义表

推荐单独维护一份道具表，例如：

```json
{
  "powerups": [
    {
      "id": "swap_ball",
      "category": "utility",
      "target": "shooter",
      "maxUsesPerRun": 1
    },
    {
      "id": "rainbow_ball",
      "category": "ball_convert",
      "target": "current_ball",
      "maxUsesPerRun": 1
    },
    {
      "id": "blast_ball",
      "category": "ball_convert",
      "target": "current_ball",
      "maxUsesPerRun": 1
    },
    {
      "id": "barrier_hammer",
      "category": "single_target",
      "target": "obstacle_ball",
      "maxUsesPerRun": 1
    },
    {
      "id": "jar_magnet_boost",
      "category": "collection_assist",
      "target": "jar_system",
      "maxUsesPerRun": 1,
      "durationTurns": 3
    }
  ]
}
```

## 9.3 推荐字段说明

- `id`
  - 道具唯一标识

- `category`
  - 道具类别
  - 推荐值：
    - `utility`
    - `ball_convert`
    - `single_target`
    - `collection_assist`

- `target`
  - 道具影响的系统目标
  - 推荐值：
    - `shooter`
    - `current_ball`
    - `obstacle_ball`
    - `jar_system`

- `maxUsesPerRun`
  - 单局最大使用次数

- `durationTurns`
  - 持续回合数
  - 仅对持续型道具使用

## 10. 交互与表现建议

### 10.1 使用时机

- 局前携带道具：在关卡开始前选择
- 局内主动道具：在当前回合发射前使用
- 失败补偿道具：在失败结算页推荐

### 10.2 表现要求

- 道具效果要短、清楚、可确认
- 道具使用后要立即反馈当前状态已变化

例如：

- `换球`：当前球与下一球快速交换
- `彩虹球`：当前球边缘变为彩虹描边
- `炸裂球`：当前球内部出现裂纹光
- `破障锤`：目标障碍球出现碎裂提示
- `缸体增强`：缸中心区域发光扩大

## 11. 平衡建议

- 单局建议最多可使用 `1~2` 个道具
- 强力道具不应连续叠加
- 后期困难关优先投放“修正型”和“辅助型”道具，而不是清屏型道具
- 不要让道具成为玩家唯一有效策略

## 12. 推荐实现顺序

建议程序实现顺序：

1. `swap_ball`
2. `rainbow_ball`
3. `blast_ball`
4. `barrier_hammer`
5. `jar_magnet_boost`
6. `failureProtection`

## 13. 一句话结论

这套系统的核心不是“帮玩家过关”，而是：

- 在玩家卡关时给出一次可理解、可操作、可翻盘的机会

对于当前项目，最推荐优先落地的是：

- `换球`
- `彩虹道具球`
- `炸裂道具球`

在此基础上，再补：

- `破障锤`
- `缸体吸附增强`
