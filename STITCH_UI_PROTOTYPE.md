# 游戏界面 UI/交互布局原型图

## 1. 使用说明

本文档用于给 Stitch AI 生成《玻璃弹珠泡泡龙》游戏界面设计图。

目标是生成一套：

- 移动端竖屏
- 可直接看出玩法结构
- 强调玻璃弹珠与陶瓷缸材质差异
- 兼顾 HUD、主战场、中央炮台、左右收集缸与结算层

建议在 Stitch AI 中优先生成以下 3 个画面：

1. `主游戏界面`
2. `胜利结算弹层`
3. `失败结算弹层`

---

## 2. 设计目标

这不是传统泡泡龙 UI，而是“上方消除 + 下方接珠”的双层结构。

设计必须突出：

- 上方珠阵是主玩法区
- 中下部中央炮台与瞄准线是操作核心
- 炮台左右两侧的彩色陶瓷缸是新机制亮点
- 掉落珠进入缸中间才算成功收集
- 缸口边缘需要明显表现为会反弹的厚陶瓷边

整体感觉应是：

- 通透的玻璃珠
- 光泽的陶瓷缸
- 明亮但不廉价
- 休闲、精致、带一点高级玩具感

---

## 3. 目标设备与比例

- 设备：手机竖屏
- 建议比例：`9:16`
- 安全区：顶部状态栏和底部手势区留白
- 设计重心：中上区域偏大，底部缸体区必须完整展示

## 5. 主界面组件说明

### 5.1 顶部 HUD

包含：

- 返回按钮
- 当前关卡编号
- 分数
- 双目标栏
- 剩余发射数

设计要求：

- 信息层级清晰，不要拥挤
- 目标栏可做成双行卡片
- 剩余发射数应比普通分数字段更醒目

### 5.2 珠阵战场区

要求：

- 占屏幕最高视觉权重
- 珠子布局要有明显上下层级与悬挂感
- 让玩家一眼能看出可断开的桥接结构
- 危险线使用半透明发光细线，不要太厚

珠子视觉：

- 玻璃弹珠质感，边缘高光明显
- 每种颜色都要有清晰轮廓，不要因为透明材质导致辨色差
- 消除前后要有轻微发光和体积感
- 特殊技能球应带有内嵌图标或特殊光晕，例如星芒、裂纹光、彩虹芯

### 5.3 瞄准与发射区

组件：

- 底部中央炮台
- 当前珠
- 下一珠
- 瞄准虚线
- 墙面反弹路径
- 幽灵落点预览

设计要求：

- 炮台位于底部中轴，必须是整个下半区的视觉中心
- 当前珠显示在炮台发射口位置
- 下一珠紧邻炮台显示，但弱于当前珠
- 当前珠比下一珠更大、更亮
- 若当前珠或下一珠为技能球，需要明显显示特殊图标和额外光效
- 瞄准路径是细虚线或发光点线
- 反弹段透明度略低于首段
- 幽灵落点使用半透明玻璃珠显示

### 5.4 底部陶瓷缸区

这是 Stitch AI 需要重点表现的创新区域。

布局要求：

- 底部中央是炮台
- 炮台左右两侧分布 4 个陶瓷缸
- 建议左侧 2 个缸，右侧 2 个缸，围绕中轴对称
- 缸体要明显低于珠阵战场，但不能弱于炮台存在感

每个缸需要具备：

- 明确颜色识别
- 厚边陶瓷缸口
- 缸内中心收集区域
- 缸壁内侧阴影
- 可容纳若干已收集玻璃珠的空间感

视觉原则：

- 缸口边缘要明显厚一些，暗示“会反弹”
- 缸中间要更开阔、更明亮，暗示“这里是有效收集区”
- 缸体颜色应该饱和但不刺眼
- 陶瓷材质和玻璃珠材质必须有明显区分

### 5.5 技能球显示规则

当前版本不设置底部技能区域。

技能以“特殊技能玻璃球”的形式存在于上方珠阵和炮台发射区：

- 上方珠阵中会混入少量特殊技能球
- 特殊技能球掉落后，直接在炮台中生成对应技能球
- 炮口当前珠和下一珠位置都可能显示技能球
- 技能球必须通过颜色、图标、内发光或边缘轮廓与普通球明显区分
- 不要用独立的技能按钮、能量条或技能槽占用底部空间

## 6. 交互原型说明

### 6.1 瞄准交互

- 玩家按住并拖动底部中央炮台区域开始瞄准
- 出现实时路径虚线
- 路径最多展示 2 次墙面反弹
- 终点显示幽灵玻璃珠

### 6.2 发射交互

- 松手后当前珠快速发射
- 当前珠槽位由下一珠补位到炮口
- 下一个新珠从炮台侧边轻微滑入
- 若玩家通过掉落获得技能球，则下一发位置直接切换为对应技能球

### 6.3 消除反馈

- 形成三消时，匹配珠子短暂高亮后碎光消失
- 若发生断层掉落，上方区域留下明显断开感
- 大掉落时让多个珠子向下落入缸区

### 6.4 缸体收集反馈

- 掉落珠砸到缸口边缘时，表现出清脆反弹
- 掉落珠进入缸中心时，快速下沉并出现收集高亮
- 同色进入对应缸时，缸体边缘可短暂发光一圈
- 掉落珠经过炮台两侧落入缸区时，炮台不遮挡下落路线和视觉判断

### 6.5 技能球掉落反馈

- 特殊技能球从珠阵中掉落时，应有比普通珠更明显的视觉提醒
- 技能球不进入普通缸体计数，而是表现为被“回收到炮台”
- 炮台在接收技能球时出现短暂吸附或能量汇入动画
- 下一发位置切换为技能球后，要有清晰确认反馈

### 6.6 失败和危险反馈

- 珠阵接近危险线时，危险线脉冲闪烁
- HUD 出现轻微红色预警
- 不要全屏警报，避免喧宾夺主

### 6.7 应用切后台处理

- 当前版本不设置单独暂停按钮
- 当应用切后台、失焦或被系统中断时，自动冻结当前局面
- 恢复时可直接继续，或弹出轻量“继续游戏”遮罩
- 该机制不需要在主界面上占用额外按钮位置

## 8. Stitch AI 主界面生成提示词

### 8.1 中文提示词

请设计一张竖屏手机游戏主界面，类型是创新型泡泡龙消除游戏。游戏主题是玻璃弹珠和陶瓷缸。屏幕上半部分是经典泡泡龙珠阵战场，珠子是透明有高光的玻璃弹珠，排列成六边形网格，局部有悬挂和桥接结构，方便产生断层掉落，普通珠子为红蓝绿黄四色，并混入少量特殊技能玻璃球，技能球要有明显图标或特殊光晕。中下部是发射与接珠复合区域，底部正中间必须有一个精致的玻璃弹珠炮台，作为主要操作中心，炮口位置显示当前弹珠，炮台旁边显示下一个弹珠，并有虚线瞄准轨迹、墙面反弹路径和半透明幽灵落点。如果掉落技能球，则下一发会在炮台中显示对应技能球。炮台左右两侧分布 4 个不同颜色的陶瓷缸，建议左 2 右 2 对称布局，缸口厚实、可反弹，缸内中心区域明亮，暗示掉落玻璃珠落入中间才算正确收集。不要设计单独的底部技能按钮区域，也不要出现单独暂停按钮。顶部 HUD 只显示关卡编号、分数、目标和剩余发射数。整体风格明亮、精致、通透、带高级玩具感，材质对比明显，玻璃珠清澈，陶瓷缸有釉面质感，界面结构清晰，适合移动端休闲游戏。

### 8.2 英文提示词

Design a polished portrait mobile game interface for a modern bubble shooter puzzle game with a premium glass marble theme. The upper half of the screen is a hex-grid marble battlefield with transparent glossy marbles in red, blue, green, and yellow, including hanging clusters and bridge-like structures that can create big falling collapses. Mix a few special skill marbles into the board, and make them visually distinct with symbols or inner glow while keeping the same glass material language. The lower middle area must feature a central marble cannon as the main control focal point. Show the current marble in the cannon, the next marble beside it, a dotted aiming trajectory, wall bounce prediction lines, and a translucent ghost landing marble. If a skill marble has been collected from a fall, the next marble shown near the cannon should visibly become that special skill marble. Around the cannon, place four colored glossy ceramic jars, arranged symmetrically with two jars on the left and two jars on the right. The jar rims should look thick and bouncy like ceramic, while the center of each jar should feel like a valid collection zone. Do not place a separate bottom skill bar, skill slots, or a pause button. Add a clean but expressive top HUD with level number, score, mission goals, and remaining shots. The style should be bright, elegant, playful, premium, toy-like, and highly readable, with a strong contrast between transparent glass marbles and glossy ceramic jars.

## 9. 胜利结算弹层原型

### 9.1 布局草图

```text
┌──────────────────────────────────────┐
│            胜利背景虚化              │
│                                      │
│          ★★★ 或 ★★☆                 │
│             关卡完成                 │
│                                      │
│        本局分数：12840               │
│        收集蓝珠：6 / 6               │
│        同色入缸奖励：+2400           │
│                                      │
│     [继续下一关]   [重新挑战]        │
│                                      │
└──────────────────────────────────────┘
```

### 9.2 设计要求

- 胜利弹层不应完全挡住背景
- 背景能隐约看到掉落珠和缸体
- 继续下一关按钮是主按钮
- 分数和收集成果要突出

### 9.3 Stitch AI 提示词

设计一个竖屏手游胜利结算弹窗，背景是虚化后的玻璃弹珠泡泡龙游戏界面。中央弹层精致、明亮、带轻微玻璃或陶瓷装饰感，显示星级评价、关卡完成标题、本局分数、收集到的指定颜色玻璃珠数量、同色入缸奖励，并有“继续下一关”和“重新挑战”两个按钮。整体风格轻快、精致、有奖励感。

## 10. 失败结算弹层原型

### 10.1 布局草图

```text
┌──────────────────────────────────────┐
│            失败背景虚化              │
│                                      │
│             差一点成功               │
│                                      │
│        未完成目标：蓝珠 4 / 6        │
│        珠阵已触碰危险线              │
│                                      │
│      [再次挑战]   [返回关卡]         │
│                                      │
└──────────────────────────────────────┘
```

### 10.2 设计要求

- 失败弹层不要太重，不做强烈惩罚感
- 重点传达“还差一点”
- 失败原因要明确
- 再次挑战按钮为主按钮

### 10.3 Stitch AI 提示词

设计一个竖屏手游失败结算弹窗，背景是虚化后的玻璃弹珠泡泡龙界面。中央弹层简洁、精致、略带安慰感，显示“差一点成功”、未完成的关卡目标、失败原因，例如珠阵触碰危险线，并提供“再次挑战”和“返回关卡”按钮。风格不要沉重，更像轻松休闲游戏中的失败提示。

## 12. 生成约束

为了避免生成偏差，请明确告诉 Stitch AI：

- 不要做成传统三消 UI
- 不要做成暗黑风或科幻风
- 不要使用厚重拟物木质风
- 不要把底部缸体做得太小
- 不要省略底部中央炮台
- 不要忽略瞄准路径和幽灵落点
- 不要让顶部 HUD 挤占过多珠阵空间
- 不要加入底部技能区域或技能槽
- 不要加入单独暂停按钮
