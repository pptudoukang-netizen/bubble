# 开发计划（更新版）

更新时间：2026-04-20  
项目路径：`E:/cocos_project/bubble`

## 1. 当前目标
在现有“上半区命中 + 下半区掉落入缸结算”玩法基础上，完成：
- 玩法稳定性收敛（掉落、缸口反弹、结算时序、音频反馈）。
- 中低端设备性能稳定（棋盘下压、大量掉落、邻居反弹时不明显掉帧）。
- HUD / Win / 选关三处星级口径统一，全部改为“目标分数驱动”。
- 可发布版本的资源加载、音频体验、流程稳定性收口。

## 2. 阶段进展概览

### 已完成（核心功能）
- 关卡选择界面接入，启动先进入选关，关卡按钮支持当前关高亮、已通关星级显示、未通关灰态。
- Win/Lose 结算链路重构：等待掉落完成后再做最终胜负判定；Lose 展示剩余步数（`left_ball_value`）。
- 玩法改为“匹配即掉落”，并同步分数口径到“即时分降低、入缸分提升”。
- 技能球参与掉落逻辑，且必须实际入缸才算生效。
- WinView 增加星级展示与弹窗动效（含星星打击感动画）。
- JarItem 规格调整：`237x230`，左右边缘碰撞区各 `40`。
- 入缸遮挡方案落地：缸体遮罩前景（`*_jar_mask`）位于掉落球上层，球到缸底后才消失。
- 命中后邻居后推反弹效果已接入，并增加了下压延迟，避免同帧偏差。
- 长屏适配已覆盖主要底部元素（缸体、炮台、待发射球等）并按“距屏幕底部固定高度”定位。
- 启动加载流程已落地：改为使用场景内 `LoadingView`，首屏加载时直接显示（无渐显）。

### 已完成（性能/稳定性）
- 掉落球渲染改为复用节点与增量回收，减少频繁创建销毁。
- 邻居反弹加入预算控制，掉落密集时降低同帧动画负担。
- 修复偶发“球卡在两缸缝隙中不消失”问题：引入吸附、超时强制收敛、卡滞检测与兜底结算。
- 掉落层级前置到固定球前方，遮罩层维持在掉落球上方，视觉层次稳定。
- 棋盘下压路径已完成首轮性能优化：棋盘泡泡节点由“整层清空重建”改为“按 cell id 复用并更新位置”。
- 调试网格层已完成节点复用，避免 `testLayer` 开启时每次下压都整层重建。
- 底部缸体层刷新条件已收紧，不再因 `board.version` 变化而无条件重绘。

### 已完成（星级/配置/工具链）
- 星级判定已从“剩余发射球数”切换为“目标分数驱动”。
- HUD `ProgressBar` 已接入实时星级进度：初始灰显，分数达到阈值后逐颗点亮。
- WinView 与选关存档已改为共用分数星级口径。
- 新增 `level.targetScore` 配置字段，并已回填 `1~20` 关的初始目标分。
- `LevelProgressStore` 新增 `completedLevels`，将“是否通关”与“星数”解耦。
- `LevelConfigLoader`、`validate-level-content.js`、`LEVEL_CONFIG_SPEC.md` 已同步支持 `targetScore`。
- 关卡配置镜像同步已完成，`assets/resources/config/levels` 与 `levels` 保持一致。

### 已完成（音频）
- 新增 `AudioManager` 与 `AudioSettingsStore`，支持：
- 背景音乐 / 音效分通道控制。
- 音量、开关、持久化存储。
- 音频资源预加载与缓存。
- 选关页背景音乐切换为 `sound/level_bg`。
- 游戏界面背景音乐切换为 `sound/game_bg`。
- 已接入的音效节点包括：选关点击、进入关卡、发射、胜利、失败。
- 掉落玻璃球与缸口反弹碰撞时，已随机播放 `ding0 ~ ding5`。

- 重点复测“发射球到位瞬间 + 大量掉落 + 邻居反弹 + 棋盘下压”峰值场景。

## 6. 验收指标（当前执行口径）
- 稳定性：无阻断性卡死、无“缝隙卡球不消失”复现。
- 体验：缸口反弹趋势符合衰减预期，不出现反弹越弹越快。
- 性能：高掉落局面下无明显长时间掉帧；棋盘下压时不再出现明显 FPS 突降。
- 星级：HUD / Win / 选关三处星级一致，按 `targetScore` 阈值实时/最终正确点亮。
- 音频：选关页 / 战斗页 BGM 切换正确，缸口碰撞随机 `ding` 正常触发且无连爆失控。
- 适配：长屏设备底部关键元素位置正确，交互可用。

## 7. 风险与应对
- 风险：参数联动多，局部改动容易影响整体手感。  
  应对：建立参数组合白名单并固定回归关卡集。
- 风险：`targetScore` 初始值沿用旧推导目标分，后续可能仍需人工微调。  
  应对：按关卡难度分段回归，优先校正前 5 关与高压关卡。
- 风险：弱机型在“发射到位 + 大量掉落 + 邻居反弹 + 棋盘下压”时仍可能抖动。  
  应对：继续做分级降载策略（预算、抽样、分帧执行）。
- 风险：音频资源仍偏临时，正式资源接入后可能需要重新平衡音量与触发频率。  
  应对：保留资源路径与音量配置入口，后续统一调音。
