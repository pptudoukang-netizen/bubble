## 3. 下一阶段计划

### Phase A：Web 手机音频收口（最高优先级）
- 在 Android Chrome / iOS Safari 实机验证：
- 首次触摸后 BGM 是否恢复
- SFX 是否可稳定播放
- 切后台返回后是否仍有声音
- 若仍有无声场景，补充平台分支兜底策略并记录最小复现步骤。

### Phase B：玩法与反馈回归
- 回归计分逻辑：确认发射/附着/消除不加分，仅入缸加分。
- 回归危险线表现：阈值触发准确，抖动强度和显示时机符合预期。
- 回归缸口与入缸音效：碰撞每次播、入缸底 `ding0` 播放稳定，无爆音。

### Phase C：配置与资源一致性收口
- 在 Cocos Editor 刷新资源并确认 `level_021~030` 的 `.meta` 自动生成。
- 验证 `assets/resources/config/levels` 的新增关卡可被扫描并进入选关。
- 继续维持根目录 `levels` 与资源目录镜像一致。

### Phase D：发布候选准备（RC）
- 完成全链路冒烟：
- 启动加载 -> 选关 -> 地图翻页 -> 进关 -> 胜/负结算 -> 返回选关
- 输出 RC 检查清单：
- 已知问题
- 风险等级
- 回滚方案

### Phase E：好友排行榜 V1（开放数据域）
- 首版范围仅包含两个榜单：
- 关卡排行（按 `max_pass_level` 查询好友最大闯关数）
- 总分排行（按 `total_score` 查询好友累计总分）
- 主域完成上报：
- 关卡胜利后更新 `max_pass_level`（仅提升不回退）
- 总分变化后更新 `total_score`
- 开放数据域完成拉取与绘制：
- 接收 `show_progress_rank` / `show_total_rank` 消息
- 拉取好友云存储并按规则排序后渲染到 `sharedCanvas`
- 明确本阶段不做：
- 世界排行榜
- 群排行榜
- 周榜/月榜
- 服务端反作弊

- 流程稳定：不出现阻断性报错，选关与进关链路稳定。
- 表现一致：关卡按钮状态、危险线、底部道具行为符合需求。

- 音频正确：

- Web 实机：
- 手机浏览器首次触摸后可正常出声
- 后续交互音效稳定

## 6. 风险与应对
- 风险：手机浏览器音频策略差异大，单机型通过不代表全面通过。  
  应对：按浏览器/系统分组记录验证矩阵，先覆盖主流机型。
- 风险：地图和关卡资源持续新增，易出现路径/元数据遗漏。  
  应对：每次同步后固定执行资源扫描与可加载性检查。
- 风险：本轮改动面广，回归遗漏概率高。  
  应对：固定冒烟路径与高风险用例（翻页、入缸、结算、返回）。
- 风险：开放数据域与主域消息协议不一致导致榜单空白。  
  应对：先冻结消息协议字段，再联调；增加空数据与异常数据兜底渲染。
